## 项目定位（说清楚是什么、不是什么）

- **是：** 一个 WorkBuddy 桌面红绿灯，Python 零依赖，通过读 JSON 文件显示状态
- **不是：** 通用 AI 工具状态监控器，也没有接入 Claude Code/Codex 等工具的 hook

> 🟡 黄灯 = 干活中
> 🔴 红灯 = 需要我介入

>
> 底层原理很简单：WorkBuddy 在干活时写一个 JSON 状态文件，红绿灯每秒读一次然后更新显示。灵感来自 GitHub 上的 AI Light 项目，但省掉了 Electron 那套重的东西。

### 知乎（如果发长文）

> ## 给 WorkBuddy 写了一个桌面红绿灯
>
> 灵感来自 GitHub 上的 AI Light 项目——它在桌面上用红绿灯显示 AI 编程助手的工作状态。但那个项目目前只有 Windows 版，而且需要装 Electron。
>
> 我照它的状态模型（绿=就绪、黄=工作中、红=需介入），用 Python 的 tkinter 重新实现了一个，依赖为零，macOS 直接跑：
>
> - 半透明悬浮窗，始终置顶，可拖拽
> - WorkBuddy 干活时自动写入状态文件，红绿灯轮询响应
> - 也支持手动点击切换和命令行控制
> - 2 分钟超时自动回退，防止卡在错误状态
>
> **限制：** 目前只适配了 WorkBuddy。如果想接 Claude Code 之类，需要自己在 hook 里写同一个 JSON 文件。
>
> 代码和文档都放 GitHub 了 👇
>
